# 水卢高报引擎 — 软件宪法 v1.0

**生效日期：2026-04-01**
**起草者：全角色审查委员会（刁钻客户 / 顶级工程师 / 营销大师 / CTO / 高考规划师 / HR / 记忆官）**

---

## 第一章：产品使命与不可逾越的边界

### 1.1 使命

水卢高报引擎的唯一使命是：**帮助中国高考生，以最低信息成本，做出最优志愿填报决策。**

### 1.2 绝不允许（宪法红线）


| 红线                       | 后果              |
| ------------------------ | --------------- |
| 推荐专科/高职院校给本科考生           | 用户信任崩塌，退款率 >50% |
| 报告显示学校数量 <30 所（正常数据省份）   | 产品失去基础价值        |
| 支付后用户看到的数据与未支付完全相同       | 违反付费契约，必遭退款     |
| 系统在未支付状态下给出完整报告          | 损害核心商业模型        |
| alert() 或系统弹窗打断用户操作流     | 极差体验，直接放弃       |
| 硬编码 IP 地址或 .com 域名在生产代码中 | 域名变更导致系统崩溃      |

## 第二章：算法宪法（核心排名与推荐规则）

### 2.1 数据池纯洁性（P-系列过滤，按顺序执行）

**P0：选科匹配**

- 物理生：排除 `subject_req IN ('历史类','文科','历史')`
- 历史生：排除 `subject_req IN ('物理类','理科','物理')` + P6 全国仅招物理专业剔除

**P1：位次有效性**

- 只使用 `min_rank IS NOT NULL AND min_rank > 0` 的记录

**P2：年份范围**

- 只使用 2017 年至今的数据

**P3：批次过滤（SQL层）**

- 排除：`提前批, 艺术, 专科, 高职, 专项, 预科, 蒙授, 民航飞行`

**P5：Python层批次兜底**

- 与P3相同关键词，对 SQL 漏网鱼进行再过滤

**P5b：校名专科兜底（宪法新增 2026-04-01）**

- 校名含以下关键词 **且** 校名不含"大学"者，一律排除：
  - `高等专科学校, 职业技术学院, 高职学院, 职业学院, 专科学校`
- 保留例外：`职业技术大学`（已升格为本科，合法推荐）

**P6：历史池全国物理专业过滤**

- 某(学校,专业)组合在全国范围内只招物理生，从历史生推荐中彻底剔除

### 2.2 概率计算

**档位定义（宪法 2026-04-01 修订）**


| 档位  | 概率范围          | 标签   | 含义               |
| --- | ------------- | ---- | ---------------- |
| 极冲  | 10% ≤ p < 25% | `极冲` | 历史上偶有录取，概率低但值得列出 |
| 冲   | 25% ≤ p < 55% | `冲`  | 有竞争性但存在风险        |
| 稳   | 55% ≤ p < 82% | `稳`  | 主力志愿区间           |
| 保   | p ≥ 82%       | `保`  | 兜底志愿             |


**重要：surge 桶的最低门槛为 10%（不是 25%）**

- 理由：高排名考生（如上海位次 < 5000）所向往的顶尖高校对其概率普遍 < 25%，若设门槛 25% 则冲桶为空，报告极度缩水。

**报告数量目标**

- **目标：96 所**（覆盖平行志愿所有可填槽位）
- 分配：冲 25 + 稳 46 + 保 25
- 每所学校最多 5 个专业跨桶共享（`_SCHOOL_CAP = 5`）
- 当 combined_96 < 30 时，自动补入高概率未入桶结果，标注"参考"

**兜底触发条件**

- 所有结果 p < 10% → 取最高 30 条标"参考"填入 safe 桶
- combined_96 ≥ 1 但 < 30 → 补齐至 30 条，标"参考"
- 正常情况下 96 所是目标，实际数量受数据量影响

### 2.3 `total_matched` 字段语义

- `total_matched`：最终展示给用户的院校专业数（combined_96 实际长度）
- `total_raw`：进入算法前的原始候选数（用于调试）
- 前端展示使用 `total_matched`

### 2.4 排序算法

综合分 = 概率×0.40 + 质量分×0.35 + 冷门价值×0.15 + 机会分×0.10

**竞争密度惩罚**：同位次区间 ±20% 内竞品过多，概率最多惩罚 12%

**热门学校流量惩罚**：近 7 天点击量 > 均值 2× 时，最多惩罚 10%

## 第三章：支付宪法

### 3.1 支付前确认门（不可绕过）

用户必须经过明确的确认步骤才能触发支付二维码，确认界面须包含：

- 套餐名称 + 价格
- 查询的省份 + 位次
- 明确的"虚拟商品·支付后不退款"警告

### 3.2 禁止自动创建订单

- 打开 PayModal 时，**不得**自动调用 `/api/payment/create`
- 只有用户点击"确认，显示支付码"后才触发
- 切换套餐计划时，重置状态，**不得**自动发起新订单

### 3.3 重复支付防护

- 检测 localStorage 中已有 `gaokao_order` 时，显示明显警告横幅
- 文案："检测到已有订单，避免重复支付"

### 3.4 退款不支持声明

- 支付码界面和成功界面均须标注："虚拟商品·解锁后不支持退款"
- 禁止出现"不满意可联系退款"类文案

### 3.5 付费墙完整性

- 未付费用户：只有第 1 个"稳"档学校显示完整数据（Trust Anchor）
- 其余全部模糊化：概率显示为区间、专业名用 * 代替
- 内部测试账号（admin token）不得在生产路径绕过付费墙

## 第五章：前端 UX 宪法

### 5.1 禁止使用 alert() / confirm()

- 一律使用 in-page toast（浮动提示条）
- toast 持续时间：2500ms
- toast 位置：当前操作卡片内或页面顶部

### 5.2 加载状态

- 每个异步操作必须有 loading 状态
- 全页加载：居中 spinner（32px）
- 按钮加载：disabled + 文案变为"加载中…"

### 5.3 错误处理

- 401 → 清除 token → 跳转 /login
- 网络错误 → toast 提示"网络异常，请重试"
- 不允许空白白屏无提示

### 5.4 分享/推荐文案（标准版）

```
我在用「水卢冷门高报引擎」填报高考志愿，位次→录取概率→冷门宝藏院校一键搞定，
比机构便宜多了！用我的专属链接注册还有优惠 👉 https://www.theyuanxi.cn/?ref=XXXX
```

### 5.5 数据来源声明（必须显示）

"以上数据基于2017–2025年公开录取数据，预测含不确定性。请以官方招生简章为准做最终决策。"

### 6.1 奖励规则

- 每邀请 1 名付费用户 → 邀请人获得 **7天** 免费会员
- 奖励叠加，无上限

| 累计邀请 | 里程碑    |
| ---- | ------ |

### 6.3 转化率计算

## 第七章：数据质量宪法

### 7.1 数据年份

- 使用 2017 年至今，共 8 年历史
- 至少需要 **3年** 数据才进行概率回归（不足3年标注"数据有限"）
- 数据每周更新一次（会员专属）

### 7.2 数据库索引（必须存在）

```sql
ix_users_referred_by, ix_users_is_paid, ix_users_sub_end,
ix_users_created_at, ix_orders_user_id, ix_orders_product_type,
ix_orders_pay_time, ix_events_user_id, ix_events_name_time
```

### 7.3 禁止硬编码

- 禁止在代码中出现 IP 地址（如 `43.143.206.19`）
- 禁止出现旧域名（如 `theyuanxi.com`）
- 所有域名引用必须使用环境变量 `SITE_URL`

## 第八章：运维宪法

### 8.1 进程管理

- 前端（Next.js port 3000）和后端（uvicorn port 8000）均由 PM2 管理
- 禁止孤儿进程（使用 `ss -tlnp | grep 800` 验证）
- PM2 必须配置开机自启（`pm2 startup && pm2 save`）

### 8.2 域名规范

- 主域名：`https://www.theyuanxi.cn`（ICP 备案主域名）
- 所有 HTTP → 301 重定向到 `https://www.theyuanxi.cn`
- `theyuanxi.cn`（无 www）→ 301 重定向到 `https://www.theyuanxi.cn`

### 8.3 安全头（必须存在）

```
Strict-Transport-Security: max-age=31536000; includeSubDomains
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
```

### 8.4 CORS

- 只允许 `https://www.theyuanxi.cn` 的跨域请求
- 开发环境额外允许 `localhost:3000`

## 第九章：测试宪法（上线前必须全部通过）

### 9.1 支付路径验证

- PayModal 打开时不自动创建订单
- 点击"确认"后 QR 码在 5 秒内出现
- 扫码支付后 3 次轮询内检测到 `is_paid=true`
- 已付费用户重开 PayModal 显示重复支付警告
- 支付成功界面无退款相关引导

### 9.2 报告质量验证

- 上海 564分 → 展示院校数 ≥ 30（目标 96）
- 报告中无专科/高职院校
- 冲桶允许极冲（p=10-24%）且标注"极冲"
- 每所学校专业数 ≤ 5

### 9.3 账户功能验证

- 微信登录用户 Dashboard 显示微信昵称而非"用户"
- 手机注册用户 Dashboard 显示脱敏手机号
- 会员到期时间正确显示
- 邀请码存在且邀请链接可复制

### 9.4 域名验证

- `http://theyuanxi.cn` → 301 → `https://www.theyuanxi.cn`
- `https://theyuanxi.cn` → 301 → `https://www.theyuanxi.cn`
- `https://www.theyuanxi.cn/api/health` → `{"status":"ok"}`
