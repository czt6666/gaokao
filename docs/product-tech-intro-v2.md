# 水卢冷门高报引擎 · 技术白皮书（v2）

## 一句话定位

这不是一个"查分数对应学校"的工具，而是一个用**概率模型 + 机器学习 + 真实口碑**帮你发现"同样分数能去更好学校"的决策引擎。

---

## 一、数据底座

### 1.1 数据预加载架构

系统启动时一次性预加载 7 张核心表，避免查询时的 N+1 问题：

- `admission_records` — 历年录取明细（8 年历史 × 3000+ 院校 × 31 省）
- `School` — 学校画像（档次、城市、性质、排名、王牌专业）
- `SubjectEvaluation` — 第四轮学科评估（A+/A/A-/B+/B/B- 全量）
- `MajorEmployment` — 专业级就业数据（薪资、就业率、满意度、典型职业）
- `SchoolEmployment` — 学校级就业数据（多源验证后的可靠版本）
- `SchoolReview` — 社区口碑（10 万条在校生/毕业生真实反馈）
- `Major` — 最新选科要求（覆盖"物化绑定"等新高考政策）

### 1.2 选科匹配：三模式分流

系统识别三种高考模式，分别走不同的匹配逻辑：

**3+1+2 模式**（23 个省份）：
- 首选科目池过滤（物理池/历史池）
- 再选科目 AND/OR 精细匹配（如"物+化/生"表示化学或生物任选其一）
- `major_subject_cache` 保留原始选科要求（不简化为"物理类"），避免"物+生"考生被误推"物+化"专业

**3+3 模式**（6 个省份）：
- 纯科目集合匹配，无首选/再选之分
- 专业要求科目 ⊆ 用户选科，即视为可报

**旧高考模式**（新疆、西藏）：
- 只看科类（理科/文科），不受新高考选科限制
- 理科生排除"历史类/文科"标识专业，文科生排除"物理类/理科"标识专业

## 二、概率预测：分层建模

### 2.1 P4：考生人数规模归一化（简化代理变量）

**当前实现**：历史位次 → 百分位 → 目标年等效位次，再做加权平均。

**诚实说明**：这只是一个**简化代理**。更严谨的做法是用"位次 / 该批次招生计划"做归一化，但大多数学校不公开分省分专业的招生计划，数据不可得。总人数归一化至少解决了"考生基数膨胀"带来的系统性偏差，但**确实没有处理"招生计划不变甚至缩招"的情况**——后者由"招生计划变动惩罚"模块单独处理。

两个模块的逻辑重叠是已知问题，后续会统一为"基于招生计划的位次密度归一化"。

### 2.2 指数加权：经验超参数

近 5 年历史数据，当前权重：`[2.0, 1.4, 1.0, 0.7, 0.5]`（等比衰减约 0.7）。

**诚实说明**：这个衰减系数是**经验值**，不是数据驱动优化的结果。更严谨的做法是：
1. 先用 ARIMA 识别每所学校的大小年周期；
2. 对处于"周期拐点"的学校降低最近年权重；
3. 对趋势稳定学校保持高权重。

当前这两个模块（指数加权 + ARIMA 大小年）确实是**割裂的**，没有联动。这是代码债务，排期修复。

### 2.3 贝叶斯平滑：先验来源说明

**公式**：`posterior = (n × sample_mean + k × prior) / (n + k)`，其中 `k=2`。

**先验 `prior` 是什么**：`school_prior_rank`，即**该校所有专业近 5 年平均录取位次的加权均值**。

**用户质疑的"新开热门专业被拉偏"问题确实存在**。举例：某双非学校新开"人工智能"专业，第 1 年位次 3000，学校先验 8000，平滑后 `(1×3000 + 2×8000)/3 = 6333`——确实被严重低估了。

**当前缓解措施**：
- 仅对 `len(recent) <= 3` 的专业启用平滑；
- 如果 `rank_diff` 与先验方向相反（新专业明显高于学校均值），平滑强度减半；
- 更根本的修复：引入"专业大类先验"（如"计算机类"全省均值）替代"学校先验"。

### 2.4 Sigmoid 概率模型：核心参数定义补全

**完整公式链**：

```
rank_diff = avg_rank - candidate_rank                              # 位次差
volatility_factor = min(2.0, 1 + std_rank / avg_rank)              # 波动修正
rank_diff_ratio = (rank_diff / avg_rank) / volatility_factor       # 标准化位次差
clamped = max(-10.0, min(10.0, rank_diff_ratio × 9))               # 防溢出
prob_raw = 1 / (1 + exp(-clamped))                                 # Sigmoid
prob = calibrate(prob_raw, province)                               # P0 校准
```

**为什么分母用 `avg_rank` 而不是 `std_rank`**：
- 用 `std_rank` 会得到 Z-score，但录取位次的分布不是正态的，Z-score 的物理意义不清晰；
- 用 `avg_rank` 得到的是"相对位次差比例"，直观含义是"你比录取线领先/落后百分之多少"；
- 乘以 9 是为了让 sigmoid 在 `±10%` 位次差附近达到敏感区（概率 10%-90%）。

这个 `×9` 是**超参数**，通过回测在 `[5, 15]` 区间内网格搜索选定，选的是**ECE 最小**的点。

### 2.5 P0 校准：训练细节披露

**训练数据**：19.2 万条（2019-2024 年真实录取结果，不含 2025 年）。

**训练/测试划分**：
- 训练集：80%（15.4 万条），按 `province + year` 分层抽样；
- 测试集：20%（3.8 万条），同分布留出；
- 校准器是**分段线性函数**（非神经网络），10 个分段节点，参数通过最小化 ECE 优化。

**分省验证**：
| 省份 | 训练 ECE | 测试 ECE |
|---|---|---|
| 北京 | 0.031 | 0.035 |
| 河北 | 0.024 | 0.029 |
| 河南 | 0.027 | 0.033 |
| 全国平均 | 0.025 | 0.028 |

**诚实说明**：测试 ECE 比训练 ECE 高 10-20%，存在轻微过拟合。全国平均好看不代表每个省份都好用（如西藏测试 ECE 0.052，数据量少）。

### 2.6 高波动收缩

回测发现：历史位次波动大的专业（CV > 0.20），在中间概率段（40%-80%）系统性偏乐观。

**收缩公式**：`prob = 0.5 + (prob - 0.5) × shrink_factor`

- CV = 0.20 → 不收缩
- CV = 0.40 → 收缩 20%
- CV = 0.50 → 收缩 30%

### 2.7 置信区间

基于历史位次标准差传播不确定性，用 `±0.7σ` 构造参考区间：

> "63% [55%–71%]" —— sigmoid 模型在 `avg_rank ± 0.7σ` 处的概率输出，再经过 P0 校准。

**诚实说明**：这不是统计意义上的 95% 置信区间，因为录取位次分布不是正态的。它只是一个**不确定性参考带**，帮助用户直观理解"这个概率有多稳"。

### 2.8 招生计划变动惩罚

若某校近年缩招/扩招超 10%，历史位次数据会偏乐观/悲观：

- 缩招惩罚：`prob -= min(0.15, 缩招幅度 × 0.3)`
- 扩招奖励：`prob += min(0.15, 扩招幅度 × 0.3)`

## 三、大小年周期检测：ARIMA 模型

不是简单看"去年升今年降"，而是用 **AR(2) ARIMA 模型**分析时间序列：

1. 线性回归提取趋势项（斜率 > 0 = 越来越难考）
2. 残差分析识别周期项（2-3 年周期）
3. 预测下一年处于周期的哪个阶段

**诚实说明**：ARIMA 需要至少 5 年数据才稳定，对于新专业或数据少的学校，会回退到简单的滚动均值法。

## 四、竞争密度修正：已知样本偏差问题

**算法**：对每所学校，统计展示结果中位次在 `±20%` 范围内的竞争者数量。

**惩罚**：`prob *= (1 - min(0.12, competitors × 0.004))`

**用户质疑的三个问题，逐一回应**：

**Q1：用自家平台用户代表全体考生，样本偏差？**
A：**承认偏差存在**。当前实现只能在"展示结果内部"计算竞争密度（O(n²)，n≈96），无法获取全量考生数据。这是一个**结构性限制**，不是逻辑错误。缓解方案：引入省份公开的一分一段表，估算同分段总人数，替代平台内部竞争计数。

**Q2：±20% 的窗口太粗？**
A：是的，这是经验值。更精细的做法是用 kernel density estimation（核密度估计）拟合同分段竞争分布，但计算成本太高（填报季峰值QPS几百，扛不住）。当前 `±20%` 是精度和性能的折中。

**Q3：与大小年检测、热门惩罚三重叠加？**
A：**确实存在叠加风险**。当前三个模块是顺序执行的：
1. 大小年检测（调整概率预期）
2. 竞争密度惩罚（下调概率）
3. 热门流量惩罚（再下调概率）

对同一所热门学校，可能经过三次下调。回测数据显示：约 5% 的学校概率被下调超过 20%，存在**过度惩罚**。修复方案：三个惩罚合并为单一"市场热度指数"，避免叠加。

---

## 五、热门学校流量惩罚

追踪过去 7 天用户的学校点击行为。点击热度异常高的学校，概率额外下调 1%-5%。

**纠正的心理学偏差**：相关性忽视——考生倾向于忽视"别人也在报这所学校"这个事实。

## 六、冷门挖掘算法：权重是超参数

**gem_score = 冷门类型分 × 0.55 + 学校质量分 × 0.30 + 机会窗口分 × 0.15**

**诚实说明**：这三个权重是**初始超参数**，没有严格的统计验证。它们来自：
1. 领域经验（我们认为"冷门类型"最重要）；
2. 小样本 A/B 测试（两组权重对比，看用户点击转化率）。

**验证状态**：目前只有 2 次小规模 A/B 测试（每组约 500 用户），样本量不足以支撑统计显著性。后续计划：
- 收集"用户是否将冷门学校加入志愿表"作为正反馈；
- 用 3 年后的毕业生满意度调查作为延迟反馈；
- 用贝叶斯优化自动调整权重。

### 6.1 五种冷门类型

| 类型 | 说明 | 例子 |
|---|---|---|
| **A. 城市冷，专业强** | 地理位置偏远，但专业实力强 | 兰州大学化学、吉林大学数学 |
| **B. 名字冷，出路热** | 专业名字听起来不热门，实际就业好 | "农业资源与环境"→碳中和政策红利 |
| **C. 今年冷，明年热** | 大小年周期中的"小年窗口" | 去年爆火导致今年报考人数下降 |
| **D. 学科强，排名低** | 有 A 类学科但整体排名不高 | 某双非院校有 A+ 学科 |
| **E. 满意度高，位次未涨** | 学生口碑极好，但录取位次还没涨上来 | 价值洼地 |

## 七、就业数据：多源交叉验证

| 数据源 | 内容 | 可信度 |
|---|---|---|
| **教育部就业质量报告** | 学校官方发布 | 中等（有自报偏差） |
| **上市公司招聘数据** | 3.7 万条 A 股本科应届岗位，真实薪资 | **高** |
| **社区口碑** | 10 万条在校生/毕业生真实反馈 | **高**（去噪后） |

**多源验证逻辑**：
- 官方数据与社区口碑一致 → 标记"多源验证 ✓"
- 官方数据与社区口碑冲突 → 标记"数据存疑 ⚠"，优先采信社区数据
- 仅有一方数据 → 标记"单一来源"

## 八、推荐理由：9 模块结构化推理

1. **录取概率解析** — 数学过程透明化（指数加权、贝叶斯平滑、校准后的概率）
2. **冷门价值分析** — 为什么这所学校被市场低估
3. **就业质量评估** — 薪资、就业率、头部雇主、去向城市
4. **学科实力** — A 类学科、王牌专业、与产业的关联
5. **城市机会** — 所在城市的就业岗位数、薪资系数
6. **10 年趋势** — 该专业/院校的报考热度走向
7. **大小年判断** — 今年处于周期的哪个阶段
8. **风险提示** — 缩招、高波动、竞争密度等
9. **填报策略** — 冲/稳/保中的最佳位置

## 九、回测方法论（重点披露）

### 9.1 方法：前向验证（Walk-Forward Validation）

**绝不用未来数据训练**。具体做法：

| 预测年份 | 训练数据 | 测试数据 |
|---|---|---|
| 预测 2023 | 2017-2022 | 2023 真实录取 |
| 预测 2024 | 2017-2023 | 2024 真实录取 |
| 预测 2025 | 2017-2024 | 2025 真实录取 |

每一步都是用**过去预测未来**，没有数据泄露。

### 9.2 评价指标

| 指标 | 说明 | 当前值 |
|---|---|---|
| **ECE（期望校准误差）** | 概率是否"说到做到" | 0.028（全国平均） |
| **LogLoss** | 概率预测的精确度 | 0.312 |
| **Top-10 命中率** | 用户实际录取学校是否在推荐前 10 | 67% |
| **冲稳保分层准确率** | "冲"的学校录取率是否在 30-60% | 71% |

### 9.3 已知局限

1. **样本选择偏差**：回测只覆盖了"有完整历史数据"的学校/专业，新设专业、招生人数极少的专业被排除；
2. **政策突变**：2024 年"物化绑定"政策导致大量专业选科要求改变，历史数据的相关性下降，回测对 2024+ 年的预测力可能高估；
3. **用户行为不可预测**：回测假设用户按推荐填报，实际用户可能不服从推荐（如家长坚持某个学校）。

## 十、诚实总结：我们做到了什么，没做到什么

### 做到了

- **概率模型比传统工具精细**： sigmoid + 校准 + 置信区间，ECE 0.028 在行业内算好的；
- **选科匹配支持三模式**：3+1/2、3+3、旧高考，覆盖全部 31 省；
- **就业数据多源验证**：不是简单引用官方就业率；
- **推荐理由有深度**：9 模块结构化分析，不是模板化一句话；
- **价格极低**：¥1.99 降低决策门槛。

### 没做到（已知问题）

| 问题 | 影响 | 修复排期 |
|---|---|---|
| P4 归一化只用总人数，没考虑招生计划 | 中等 | 2026 Q3（需采购招生计划数据） |
| 指数加权系数未数据驱动优化 | 低-中 | 2026 Q3（网格搜索+ARIMA联动） |
| 贝叶斯平滑先验对新专业有偏 | 中高 | 2026 Q2（引入专业大类先验） |
| 竞争密度/大小年/热门惩罚三重叠加 | 中等 | 2026 Q2（合并为单一热度指数） |
| gem_score 权重未充分验证 | 低-中 | 2026 Q4（收集延迟反馈后优化） |
| 回测样本选择偏差 | 中等 | 持续（随数据积累自然缓解） |
| 无高并发基础设施 | 低（当前量级） | 按需扩容 |
