## 概述

本引擎的目标：**识别"被市场低估的学校/专业"**，替代"冲稳保"线差法，输出第四类——冷门宝校。

核心假设：考生/家长在填报时会系统性高估城市热度、学校排名、专业名字直观度；低估就业数据、产业走向、学科实力等硬指标。两者的落差 = 价值洼地。

算法共 **5 层架构**，每层独立产出分值，最终加权融合：

```
层 0  数据底座          229万录取记录 + 就业/学科评估/学校档案
层 1  动态冷门评分       4维公式 → ColdScore (0-100)
层 2  7类冷门检测        A~G 七种触发条件并行计算 → compound_score
层 3  产业信号层         S曲线 + AI互补性 + 委培关系表
层 4  群体智能预测       300 Agent 模拟 → swarm_consensus
层 5  自适应融合排序     stat × swarm × behavior 加权
```

## 层 1 · 动态冷门评分引擎

**文件**：`algorithms/cold_score_engine.py`

### 核心公式

```
ColdScore = 认知差距 × 0.30
          + 薪资错配 × 0.25
          + 产业动能 × 0.25
          + 供给稀缺 × 0.20
```

### ① 认知差距分 (recognition_gap) · 权重 0.30

> **逻辑**：市场报考热度 vs 真实薪资水平的差距。

- 对每个专业计算：
  - `avg_rank`：2022 年后各省平均录取位次
  - `avg_salary`：专业平均月薪
- 在全市场范围内求百分位：
  - `rank_pct` = 位次百分位（数字越大=位次越靠后=越冷门）
  - `sal_pct` = 薪资百分位（越高越好）
- 差距值：
  ```
  gap = sal_pct − (1 − rank_pct)
  # 正值 = 薪资好但录取容易 = 被低估
  # 负值 = 过热（薪资一般但分数炒得很高）
  ```
- 最终归一化：`(gap + 0.5) × 100`，区间 0–100

### ② 薪资错配分 (salary_mismatch) · 权重 0.25

> **逻辑**：薪资排名 vs 录取难度排名，差距越大越冷门。

```python
mismatch = sal_rank_pct − admit_rank_pct
score = sigmoid(mismatch × 8) × 100
```

Sigmoid(×8) 的作用是将小差异放大，让中间段更敏感。

### ③ 产业动能分 (industry_momentum) · 权重 0.25

> **逻辑**：静态词库匹配专业所属产业，给出 0-100 动能分。

**数据源**（2026-04-07 更新）：
1. A 股上市公司 2021→2024 真实招聘量变化率（2100 万条招聘数据）
2. 2025 Q1 最新岗位薪资水平
3. 国家政策文件（十四五/十五五重点产业）
4. 3 篇顶刊论文（《管理世界》《经济研究》《中国管理科学》）

**分档表**（部分摘录）：

| 动能分 | 档位 | 代表专业 |
|---|---|---|
| 90-100 | 国家战略 | 低空经济 95、核工程 92、飞行器 90、集成电路 88 |
| 82-90 | 双碳/新能源 | 新能源材料 88、储能 86、自动驾驶 88、量子 82 |
| 75-85 | AI/数字/生医 | 人工智能 82、生物医学工程 83、遥感 82、机器人 80 |
| 60-75 | 传统稳健 | 统计学 72、精算 74、电气 73、机械 68 |
| 25-55 | 供给过剩 | 土木 30、房地产 15、会计 38、金融 42、软件 48 |

未匹配专业默认 55 分。

### ④ 供给稀缺分 (supply_scarcity) · 权重 0.20

> **逻辑**：招生计划少 → 毕业生供给少 → 稀缺溢价。

```python
supply_score = 95 − log(avg_plan + 1) × 15
# 5人→90，15人→75，30人→60，100人→40
```

```

   ↓ miss
文件缓存 (.cold_scores_cache.json, TTL 3天)
   ↓ miss
实时计算 (compute_major_cold_scores)
```

## 层 2 · 七类冷门检测器

**文件**：`algorithms/hidden_gem.py`

对每所候选学校+专业并行跑 7 个独立检测器，每类产出 `gem_score (0-100)`，最终复合评分。

### A 型 · 城市冷·专业强（城市折价）

**触发条件**：
- 城市折价系数 ≥ 0.4
- 至少一个 A 级学科评估（A++/A+/A/A-）

**v2.0 关键修复**：采用**省际相对折价**。
- 哈尔滨考生报哈尔滨学校，城市"冷"对他不构成折价
- 公式：`delta = home_heat − school_heat`（家乡省会热度 − 学校城市热度）
- `delta ≤ 1` 不触发

**城市热度表**（1-10）：

```
北京/上海  10    广州/深圳  9     杭州/南京  8
成都/武汉  7-8   西安/长沙/重庆/天津  6-7
合肥/郑州/济南  6   长春/沈阳/哈尔滨  4
兰州/乌鲁木齐/呼和浩特  3    拉萨  2
```

`gem_score = 城市折价 × 100`

### B 型 · 名字冷·出路热（认知折价）

**双路径设计**：

**动态路径**（主）：层 1 的 ColdScore ≥ 65 即触发
- 65-70 低档、70-80 中档、≥80 高档

**静态路径**（描述源）：词库 `COGNITIVE_DISCOUNT_MAJORS` 提供人类可读的误解纠正
- 覆盖 50+ 专业，每条含 `{real_direction, industry_prospect, misconception, discount_level}`

**示例**：

| 专业 | misconception（误解纠正） |
|---|---|
| 地球物理学 | 以为是纯理论，其实就业于石油、地震局，薪资高 |
| 核工程与核技术 | 以为危险或冷门，实际是国家重点支持领域，薪资极高 |
| 知识产权 | 以为是细分冷门，实际是科技企业最紧缺的法务方向之一 |
| 监狱学 | 陌生专业，实际毕业后直接进编制，就业率接近100% |
| 林学 | 以为是去山里种树，实际是碳市场爆发的核心专业 |

两路径合流：动态分决定"是否冷门"，静态词库提供"为什么冷门/真相是什么"。

### C 型 · 今年冷·明年热（时机型）

**触发条件**：近 2-3 年该专业录取位次**持续上升**（数字变大 = 录取门槛降低 = 报考人数减少）。

- 总降温幅度 ≥ 8%
- `gem_score = min(95, 降温幅度 × 300)`

**v2.0 重要修复**：v1.0 逻辑反转（把"位次变小=变热"误读为"位次变小=变冷"），现已修正。位次数字变大才是触底窗口。

### D 型 · 学科强·排名低（学科强校折价）

**触发条件**：
- 软科综合排名 > 100 或无排名
- 至少一个 A 类学科评估（A++/A+/A）

**逻辑**：综合排名低拖累录取竞争度，但学科实力在该方向与顶尖校齐平 → 捡漏。

```python
gem_score = 75 + min(15, A类学科数量 × 5)
```

### E 型 · 口碑优·认知慢（口碑折价）

**触发条件**：专业满意度 ≥ 4.3/5.0（前 20% 分位）。

**逻辑**：在读学生打分高意味着教学质量和就业预期已被验证，但市场报考还没反应过来。

```python
gem_score = min(85, 满意度 × 15)
```

### F 型 · 产业上升·未来溢价（v2.0 新增）

**触发条件**：
- 专业所属产业 S 曲线分 > 0.40
- AI 互补性 ≥ -0.20（不会被 AI 替代）

```python
combined = industry_score × 0.70 + max(0, ai_comp) × 0.30
gem_score = min(95, combined × 100)
```

**AI 互补性分档描述**：
- ≥ 0.70：AI 高度互补，工程师需求随 AI 扩张而增长
- ≥ 0.30：AI 辅助提效，人类判断不可替代
- 其他：AI 影响有限，专业技能壁垒较高

### G 型 · 委培定向·就业确定（v2.0 新增）

**触发条件**：学校名+专业名命中 `KNOWN_ENTRUSTED_TRAINING` 表。

**分档表**：

| tier | 分值 | 代表 |
|---|---|---|
| guaranteed | 95 | 中央司法警官学院、中国人民公安大学 |
| tier1_tech | 90 | 福耀科技大学 |
| tier1_mil | 88 | 国防科大、海军工程大学 |
| tier1_soe | 82 | 中国石油大学+石油、哈工程+核、大连海事+航海 |
| tier1_mfg | 80 | 制造业头部定向 |
| tier_public | 75 | 南信大+气象、河海+水利、中国地质+资源勘查 |

### 复合评分（v2.0 关键改进）

旧版本取 `max(gem_scores)`，浪费了多信号共振的信息。新版本：

```python
primary = max(所有命中 gem.score)
reinforcement = Σ(g.score × 0.15 for g in 其他 gem if g.score > 30)
compound_score = min(100, primary + reinforcement)
```

多类型同时触发 → 冷门置信度叠加，避免单一信号误判。

## 层 4 · 群体智能预测

**文件**：`algorithms/swarm_predictor.py`

**核心思想**：生成 300 个虚拟考生 Agent，人格参数**从真实录取/就业数据反推**（非随机），让它们集体选志愿，涌现出的共识揭示统计模型低估的真冷门。

**性能**：< 30ms / 次查询（numpy 向量化，无 LLM 调用，零 API 成本）。

### 省份人格参数（6 维，数据驱动）

| 人格维度 | 反推数据源 | 取值范围 |
|---|---|---|
| risk_tolerance | 该省位次变异系数（越大越保守） | 0.20~0.60 |
| prestige_weight | 985/211 录取占比 | 0.35~0.85 |
| career_weight | 学校平均就业率 | 0.30~0.80 |
| city_weight | 跨省录取占比 | 0.30~0.75 |
| gem_openness | 非知名校录取占比 | 0.15~0.55 |
| postgrad_focus | 学校平均深造率 | 0.20~0.70 |

数据不足时回退到默认值（DEFAULT_PARAMS）。

### 模拟流程

```
1. 按 school_name 去重（同校多专业取概率最高一条）
2. 构建学校特征矩阵 (N × 6)
   [prob_norm, quality_norm, city_score, employ_rate, gem_norm, sentiment]
3. 生成 Agent 矩阵 (300 × 6)
   personalities = base_vec + Normal(0, 0.15)，再 clip 到 [0.05, 0.95]
4. 构建权重矩阵 W (300 × 6)，按人格维度缩放：
   prob       : 0.40（固定）
   quality    : personalities[:, 1] × 0.22  (prestige驱动)
   city       : personalities[:, 3] × 0.12  (city驱动)
   employment : personalities[:, 2] × 0.12  (career驱动)
   gem        : personalities[:, 4] × 0.10  (openness驱动)
   sentiment  : 0.04（固定）
   W /= W.sum(axis=1)
5. 打分 scores = W @ mat.T
6. 风险调整：保守 Agent 偏好高概率学校
   scores *= 1 + (risk - 0.5) × 0.20
7. Softmax(temp=0.30) → 每个 Agent 选 12 所志愿
8. 统计每所学校被选次数 → 归一化为 swarm_consensus_score (0-1)
```

**关键设计**：
- **按学校去重**避免"选票分裂"（同校多专业抢票）
- **seed = base_rank % 99991** 保证同位次可复现
- **temp=0.30** 在区分度与多样性间平衡

### 群体强推判定

```python
swarm_discovery = (30 ≤ probability ≤ 65) and (swarm_score ≥ 0.60)
```

**含义**：统计上处于"冲/稳"区间（不够稳），但群体共识极强 → 多维综合优秀但位次匹配低估。这就是算法发现的真正冷门宝校。

## 层 5 · 最终融合排序

### 自适应权重

```
_swarm_final = probability × W_stat
             + swarm_score × W_swarm
             + behavior_boost × W_beh
```

| 行为样本数 | W_stat | W_swarm | W_beh |
|---|---|---|---|
| < 20 条 | 0.65 | 0.30 | 0.05 |
| 20-100 | 0.60 | 0.25 | 0.15 |
| > 100 | 0.55 | 0.20 | 0.25 |

随用户"加入志愿表"行为数据积累，权重自动从**统计驱动**过渡到**群体+行为共同驱动**。

### 行为数据来源

```sql
SELECT COUNT(*) FROM user_events
WHERE event_type='add_to_form' AND province=:p
```

即该省用户把学校加入志愿表的次数，作为"真实选择"信号。

## 端到端调用链

```
用户输入 (province + rank + subject_choice)
    │
    ▼
候选学校集（Sigmoid录取概率 20%~95%）
    │
    ▼
层1: 每专业打 ColdScore（三级缓存）
    │
    ▼
层2: 并行跑 A-G 七类检测器
    ├─ A 省际相对城市折价
    ├─ B 动态/静态双路径认知折价
    ├─ C 位次上升触底窗口检测
    ├─ D 学科强校折价
    ├─ E 满意度口碑折价
    ├─ F 产业 S 曲线 + AI 互补性
    └─ G 委培关系表命中
        │
        ▼
    compound_score = primary + Σ reinforcement
    │
    ▼
层3: F/G 调用产业 S曲线 + AI暴露 + 委培表
    │
    ▼
层4: 300 Agent 群体模拟 → swarm_consensus
    │
    ▼
层5: 自适应融合 → _swarm_final 排序
    │
    ▼
输出四分类：冲 / 稳 / 保 / **冷门宝校**
```
